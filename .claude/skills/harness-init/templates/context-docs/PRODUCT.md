# Product

The north star and the model everything hangs off. Feedforward source for the spec generator and refine's
scope-guard: a proposed change that drifts from what's here should be flagged, not built.

## North star

- One or two sentences: what this is, the core loop, who it's for.

## It IS / It is NOT (scope edges — reject drift here)

- **It IS:** the capabilities that are in scope.
- **It is NOT:** the adjacent things this is explicitly not (the non-goals — the tempting drift).
- The tie-breaker rule when a feature is ambiguous.

## Core model

- The few central nouns/concepts and how they relate (this drives the whole UI/API).
- Key states each concept can be in.

## Locked decisions

- Decisions already made that a change must not silently reopen — each with a one-line why.

## Out of scope

- Explicit non-goals for the current phase, so a reviewer can cite them.
