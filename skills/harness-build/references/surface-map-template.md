# Surface Map Template

Output of Step E surface mapping. Persisted to `<change-state-dir>/surface-map.md` (path per
HARNESS.md). Read by all implementing agents during execution.

## Output shape

```markdown
# Surface Map: <change-name>

Generated: <ISO timestamp>

## Files touched

| File | Tasks | Package |
|------|-------|---------|
| <path/to/file-a> | 3.1, 3.4, 3.5 | <package-a> |
| <path/to/file-b> | 1.1, 1.3 | <package-b> |

## Task groups

| Group | Tasks | Nature |
|-------|-------|--------|
| 1 | 1.1–1.5 | <e.g. UI primitives> |
| 2 | 2.1–2.4 | <e.g. utilities> |
| 3 | 3.1–3.11 | <e.g. feature components> |

## Parallel clusters per group

Group 1:
  Round 1a [parallel]: 1.1 (file-a), 1.2 (file-b)
  Round 1b [serial]:   1.3 (index — depends on 1.1, 1.2)
  Round 1c [parallel]: 1.4 (file-a.spec — depends on 1.1), 1.5 (file-b.spec — depends on 1.2)

Group 2:
  Round 2a [parallel]: 2.1 (util-a), 2.3 (util-b)
  Round 2b [parallel]: 2.2 (util-a.spec), 2.4 (util-b.spec)

## Transitive dependencies

| File | Depends on |
|------|-----------|
| <path/to/composite> | <dep-a>, <dep-b> |

## Existing tests

| Test file | Covers |
|-----------|--------|
| <path/to/file-a.spec> | <path/to/file-a> |

## Loaded conventions

| Rule file | Matched because |
|-----------|----------------|
| <rules-dir>/<rule-a> | tasks mention <signal> |
| <rules-dir>/<git-rule> | always loaded |
```

## Authoring notes

- Generated by Step E (inline orchestrator work) — never edited by hand mid-build.
- If the surface changes (task added, file renamed), regenerate by deleting the file and re-running
  Step E surface mapping.
- Parallel clusters are computed **conservatively: same-file = serialize. When in doubt, serialize
  rather than parallelize.**
- **Always-loaded rules:** any rule whose heading or first paragraph references git, PR workflow, or
  pre-commit conventions. These apply to every change regardless of detected surface.
